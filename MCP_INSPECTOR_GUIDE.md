# üîç MCP Inspector Testing Guide for Task List Code Review MCP Server v0.2.1

This guide walks you through installing and using MCP Inspector to test our Task List Code Review MCP server functionality. MCP Inspector is the official testing tool for Model Context Protocol (MCP) servers.

## ü§î What is MCP Inspector?

MCP Inspector is a web-based tool that allows you to:
- Connect to and test MCP servers
- Explore available tools and their schemas
- Execute tools with custom parameters
- View tool responses and debug issues
- Understand how MCP servers communicate

- Node.js 18+ installed on your system

- Basic understanding of our project structure

## üîå Connecting Our MCP Server

1. **In MCP Inspector**, click "Add Server" or "Connect to Server"

2. **Configure Server Connection**:

## üõ†Ô∏è Exploring Available Tools

### 1. Tools Panel
In the MCP Inspector interface:
- Click on "Tools" in the left navigation
- You should see **2 available tools**:
  - `generate_code_review_context`
  - `generate_ai_code_review`

### 2. Tool Schemas

#### üìã `generate_code_review_context`
- **Description**: "Generate code review context with flexible scope options"
- **Parameters**:
  - `project_path` (‚úÖ required, string): Absolute path to project root directory
  - `scope` (üîß optional, string): Review scope - 'recent_phase', 'full_project', 'specific_phase', 'specific_task' (default: 'recent_phase')
  - `phase_number` (üîß optional, string): Phase number for specific_phase scope (e.g., '2.0')
  - `task_number` (üîß optional, string): Task number for specific_task scope (e.g., '1.2')
  - `current_phase` (üîß optional, string): Legacy phase override (e.g., '2.0'). Auto-detects if not provided
  - `output_path` (üîß optional, string): Custom output file path. Uses smart naming if not provided
  - `enable_gemini_review` (üîß optional, boolean): Enable Gemini AI code review (default: true)
  - `temperature` (üîß optional, number): Temperature for AI model (default: 0.5, range: 0.0-2.0)

#### ü§ñ `generate_ai_code_review`
- **Description**: "Generate AI-powered code review from existing context file"
- **Parameters**:
  - `context_file_path` (‚úÖ required, string): Path to existing code review context file (.md)
  - `output_path` (üîß optional, string): Custom output file path for AI review
  - `model` (üîß optional, string): Optional Gemini model name (supports aliases)
  - `temperature` (üîß optional, number): Temperature for AI model (default: 0.5, range: 0.0-2.0)

## üö® Common Issues and Troubleshooting

### Connection Issues

**‚ùå Problem**: "Failed to connect to server"
**‚úÖ Solutions**:
1. Ensure uvx is installed: `pip install uvx`
2. Verify package installation: `uvx task-list-code-review-mcp --help`
3. Check for any Python environment issues
4. üî• **New**: Verify you're using correct version: `uvx --from task-list-code-review-mcp==0.2.1 task-list-code-review-mcp`

**‚ùå Problem**: "Server process exited"
**‚úÖ Solutions**:
1. Check if all dependencies are installed
2. Verify Python version compatibility (3.8+)
3. Look at server logs in Inspector's debug panel
4. üî• **New**: Check GEMINI_API_KEY is set (required for AI features)

### Tool Execution Issues

**‚ùå Problem**: "Tool execution failed"
**‚úÖ Solutions**:
1. Verify all required parameters are provided
2. Ensure paths are absolute and exist
3. Check file permissions for output directory
4. üî• **New**: For `generate_ai_code_review`, ensure context file exists and is readable

**‚ùå Problem**: "Gemini API errors"
**‚úÖ Solutions**:
1. Set `enable_gemini_review: false` to test without AI
2. Check if GEMINI_API_KEY environment variable is set (required)
3. Verify API key has proper permissions
4. üî• **New**: Use model aliases for easier configuration (e.g., "gemini-2.5-pro" instead of full preview names)
5. Check if model supports advanced features based on model_config.json
